# CLAUDE.md — Биология · тренажёр к экзамену

Статический сайт-тренажёр (PWA) для подготовки к экзамену по биологии — мед.
специальность **«Лечебное дело»**, 126 экзаменационных билетов. Сделан для конкретного
человека. Язык интерфейса — **русский**, тон академический, текст должен легко читаться
с телефона.

> Этот проект **не связан** с другими в папке `Проект/` (NoEnergyDrink, manicure и т.д.).
> Самодостаточный, отдельный git-репозиторий.

## Живые адреса

- **Основной (доступен в РФ без VPN):** https://bognyfken.github.io/bio-exam/ — GitHub Pages
- **Запасной:** https://bio-exam.pages.dev — Cloudflare Pages (в РФ часто требует VPN)

Если и GitHub начнёт тормозить в РФ — план Б: российский хостинг (Timeweb / Yandex Object

## Стек и принципы

- **Чистая статика, без бэкенда, без фреймворков, без сборки.** Vanilla JS (ES-модули) +
  CSS + HTML. Pages раздаёт папку `public/` как есть.
- **Markdown → HTML:** библиотека `marked`, **вендорена локально** в `public/vendor/marked.min.js`
  (не CDN — сайт должен работать офлайн после первой загрузки).
- **Контент грузится в браузере через `fetch`** из `public/content/`. Парсинг — один раз при
  загрузке. Код **не зависит** от конкретных билетов: меняешь контент — всё подхватывается.
- **Прогресс** — в `localStorage` (на `file://` не работает; тестировать через http-сервер).
- **PWA:** manifest + service worker (офлайн-кэш). Иконка — двойная спираль ДНК.

## Формат контента — `public/content/konspekt.md`

⚠️ **Реальный формат ОТЛИЧАЕТСЯ от образца в `claude-code-brief.md` (§2).** Бриф описывал
разделы через H1 и подпункты-blockquote — в жизни файл размечен иначе. Парсер написан под
**реальный** файл, он источник правды:

| Элемент | Разметка | Что парсер извлекает |
|---|---|---|
| Заголовок документа | `# …` | игнорируется |
| **Раздел** | `## РАЗДЕЛ I. НАЗВАНИЕ` | имя = sentenceCase без префикса «РАЗДЕЛ X.» |
| **Билет** | `### N. Название` | `number`, `title` (markdown в title зачищается) |
| Подпункты вопроса | `**а) Заголовок.** текст…` | жирный лид-ин в начале строки → `subpoints[]` |
| 🧠 Мнемоника | строка с `🧠` | плашка `.callout-mnemo` |
| ⚠️ Ловушка | строка с `⚠️` | плашка `.callout-trap` |

Служебный раздел `## ✅ КОНСПЕКТ ЗАВЕРШЁН…` отбрасывается (нет билетов).
Структура билета после парсинга: `{ id:"ticket-N", number, title, section, subpoints[],
bodyMarkdown, bodyHtml, plainText }`. `bodyHtml` — через `marked`; мнемоники/ловушки помечаются
классами пост-обработкой HTML. `plainText` — для поиска.

**Менять формат конспекта не нужно** — пиши новый контент в той же разметке, парсер подхватит.

## Режимы (нижний таб-бар, всегда виден)

- **Главная** — % выученного, прогресс по разделам, история тестов, сброс прогресса.
- **Чтение** — билеты по разделам, сворачивание, блок вопроса (подпункты), плашки 🧠/⚠️,
  кнопка «Выучить», якоря `#ticket-N` (deep-links).
- **Карточки** — 1 билет = 1 карточка, переворот, «Знаю/Не знаю», фильтр по разделу/билету,
  перемешивание, **система Лейтнера** (3 коробки) в localStorage, клавиши ←/→/пробел.
- **Тест** — MCQ из `quiz.json`, выбор области (раздел/билет/всё) и числа (10/20/всё),
  подсветка верного/неверного, пояснения, итог с разбивкой по разделам, разбор ошибок со
  ссылками «→ Билет N в Чтении», история в localStorage.
- **Поиск** — мгновенный (debounce 150 мс) по title + подпунктам + тексту, подсветка, переход к билету.

## Тесты — `public/content/quiz.json`

Массив объектов:
```json
{ "id":"q4-1", "ticket":4, "section":"Цитология и молекулярная биология",
  "question":"…", "options":["…","…","…","…"], "answer":0, "explain":"…" }
```
- `answer` — **индекс** верного варианта в `options` (0..3). `id` уникальны.
- `section` — для отображения и группировки в итоге; должен совпадать с именем раздела из
  парсера (sentenceCase). Фильтр теста по разделу идёт по номерам билетов, так что критично
  только `ticket`, но `section` лучше держать корректным.

### Как перегенерировать тесты (при смене конспекта)

MCQ нельзя сделать механически — нужны осмысленные дистракторы. Текущие 510 вопросов
сгенерированы **8 параллельными подагентами по разделам**. Чтобы повторить:

1. Заново определить разделы и диапазоны билетов из нового `konspekt.md`
   (`## РАЗДЕЛ` → раздел, `### N.` под ним → билеты). На момент 510-вопросной версии было:

   | Раздел (display name) | Билеты |
   |---|---|
   | Цитология и молекулярная биология | 1–22 |
   | Генетика | 23–42 |
   | Индивидуальное развитие (онтогенез) | 43–52 |
   | Эволюционное учение | 53–70 |
   | Филогенез систем органов хордовых | 71–76 |
   | Биосфера и экология | 77–90 |
   | Медицинская паразитология — возбудители | 91–126 |

2. Запустить по подагенту на раздел (большие — поделить пополам). Каждый читает свой
   диапазон билетов, генерит 3–4 MCQ на билет, пишет валидный JSON-массив в `quiz_parts/partNN.json`.
   Правила: дистракторы правдоподобные из той же области; варьировать позицию ответа;
   `explain` короткий; русский, без markdown в тексте.
3. Слить части в `quiz.json`, провалидировать (уникальность id, `answer` 0..3, ровно 4
   варианта, покрытие билетов), удалить `quiz_parts/`.
4. Это машинная генерация — возможны спорные формулировки; в разборе ошибок есть ссылка на билет.

## Локальная разработка

`localStorage`/`fetch` markdown не работают по `file://` — нужен http-сервер:
```bash
python -m http.server 8765 --bind 127.0.0.1   # затем http://127.0.0.1:8765/public/  (или из public/)
# или:  npx serve public   /   npx wrangler pages dev public
```
Service worker работает на `localhost`/`127.0.0.1` (secure context). Помни про кэш SW при
повторных загрузках — для чистой проверки бампай `VERSION` в `sw.js` или отключай кэш.

## Деплой

**Основной — GitHub Pages.** `main` = исходники, ветка `gh-pages` = содержимое `public/` в
корне. После изменений:
```bash
bash scripts/deploy.sh
# = git push origin main  +  git subtree push --prefix public origin gh-pages
```
> Токен GitHub **без scope `workflow`** → GitHub Actions недоступен, поэтому ветка `gh-pages`
> (а не Actions). gh CLI авторизован на машине (keyring, аккаунт `bognyfken`).

**Запасной — Cloudflare Pages:**
```bash
set -a; . ./.env; set +a
npx wrangler pages deploy public --project-name=bio-exam --branch=main
```

## PWA / service worker — ВАЖНО

`public/sw.js` кэширует app shell и контент. **При любом изменении ассетов или контента
(`konspekt.md`, `quiz.json`, js, css) — поднять `VERSION` в `sw.js`** (`bio-v1` → `bio-v2` …),
иначе у уже установивших PWA останется старый кэш. Адрес сменился с Cloudflare на GitHub —
если ставили иконку на «Домой» со старого адреса, переставить с нового.

## Иконка

Иконка-ДНК генерируется: `node scripts/gen-icons.mjs` → пишет `public/icon.svg` и PNG в
`public/icons/`. ⚠️ Скрипт берёт `sharp` из `node_modules` соседнего проекта NoEnergyDrink
(в этом проекте своего `package.json` нет) — путь захардкожен в скрипте. Если соседнего
проекта не будет — поставить sharp локально или растеризовать SVG другим способом (есть и
ImageMagick `convert`).
