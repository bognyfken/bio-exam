# Бриф для Claude Code — учебный сайт-тренажёр «Биология: подготовка к экзамену»

> Вставь это как первое сообщение в сессию Claude Code (VS Code).
> Окружение: установлен Node.js, есть `wrangler`, в env заданы `CLOUDFLARE_API_TOKEN` и `CLOUDFLARE_ACCOUNT_ID`.

## 0. Цель

Собрать **статический сайт-тренажёр** для подготовки к экзамену по биологии (мед. специальности «Лечебное дело» / «Педиатрия», 125 экзаменационных билетов).

Источник контента — **один markdown-файл** `content/konspekt.md` (конспект по билетам). Сайт парсит этот markdown **в браузере** и строит из него четыре режима: **Чтение · Карточки · Тесты · Поиск**. Навигация — двойная: по билетам и по крупным разделам.

Деплой — **Cloudflare Pages**, прямой upload через `wrangler` (без GitHub). Главный сценарий использования — повторение **с телефона (iPhone/Safari)**, поэтому mobile-first и быстрая загрузка обязательны.

## 1. Технологии и принципы

- **Чистая статика, без бэкенда.** Никаких серверных функций, никаких внешних API в рантайме.
- **Zero-build по возможности:** простой `index.html` + `app.js` (ES-модули) + `styles.css`. Фреймворки (React/Vue) НЕ использовать. Если без бандлера совсем неудобно — допускается минимальный Vite, но это нежелательно; приоритет — простота деплоя (Pages раздаёт папку как есть).
- **Markdown → HTML:** библиотека `marked`. **Вендорить локально** в `vendor/marked.min.js` (не тянуть с CDN в рантайме — сайт должен открываться и работать офлайн после первой загрузки).
- **Контент грузится через `fetch`:** `content/konspekt.md` и `content/quiz.json`. Обновление контента = замена файла + редеплой. Парсинг markdown в структуру данных делается один раз при загрузке.
- **Mobile-first**, адаптивная вёрстка. Быстрый первый рендер. Опционально — простой service worker для офлайн-кэша.
- **Прогресс** хранится в `localStorage` (на задеплоенном сайте это работает; в локальном `file://` — нет, тестируй через `wrangler pages dev` или `npx serve`).
- Интерфейс — **на русском**. Шрифт с хорошей читаемостью кириллицы.

## 2. Контракт `konspekt.md` — СТРОГО соблюдать при парсинге

Конспект пишется по простым правилам. Парсер обязан опираться именно на них:

```markdown
# Цитология                            ← H1 = РАЗДЕЛ (крупная тема). Действует до следующего H1.

## 4. Клеточная теория                 ← H2 = БИЛЕТ. Шаблон: "## <номер>. <название>"
> а) История формирования представлений о клетке.   ← blockquote = подпункты билета (текст вопроса)
> б) Основные положения клеточной теории.
> в) Развитие клеточной теории.

Тело ответа в обычном markdown: подзаголовки ###, списки,
**жирный текст**, _курсив_, таблицы — всё допустимо.

🧠 Мнемоника пишется строкой, начинающейся с эмодзи мозга.
⚠️ Ловушка на экзамене — строкой с предупреждающим эмодзи.

## 5. Организация эукариотической клетки
> а) Клеточные мембраны.
> б) Цитоплазма.

...тело ответа...
```

**Правила парсинга:**

1. `# ` (H1) — начинает новый **раздел** (`section`). Всё до следующего H1 относится к нему.
2. `## ` (H2) — начинает новый **билет** (`ticket`). Извлекай: `number` (целое до точки) и `title` (текст после `«N. »`).
3. Идущие сразу за H2 строки-`blockquote` (`> …`) — это **подпункты билета** (`subpoints`, массив строк). Это текст самого вопроса.
4. Весь markdown после подпунктов и до следующего `##`/`#` — это **тело ответа** (`body`, markdown-строка).
5. Строки тела, начинающиеся с `🧠` → пометить как мнемонику (особый стиль). Строки с `⚠️` → пометить как ловушку (особый стиль).

**Структура данных после парсинга (ориентир):**

```js
[
  {
    section: "Цитология",
    number: 4,
    title: "Клеточная теория",
    subpoints: ["а) История формирования...", "б) Основные положения...", "в) ..."],
    bodyMarkdown: "### ...\n- ...",
    bodyHtml: "<...>",        // прогнать через marked
    plainText: "...",          // для поиска (без разметки)
    id: "ticket-4"
  },
  ...
]
```

> Если `konspekt.md` ещё нет — используй образец из §9, чтобы собрать и протестировать сайт. Структуру не меняй: настоящий файл придёт в том же формате.

## 4. Режимы и функции — детально

Верхняя панель (на мобиле — нижний таб-бар) переключает 4 режима: **Чтение · Карточки · Тест · Поиск**. Плюс стартовый экран **Главная** с прогрессом.

### 4.1 Навигация (двойная)

Боковая панель (на мобиле — выезжающий drawer / гамбургер). Две вкладки переключения вида списка:

- **По разделам:** раскрывающиеся разделы (Цитология, Генетика, Эмбриология/онтогенез, Эволюция, Антропогенез, Филогенез, Биосфера/экология, Паразитология…) → внутри список билетов раздела.
- **По билетам:** плоский нумерованный список 1…125 (можно с подзаголовками-разделами как разделителями).

Клик по билету → открывает его в режиме **Чтение** и скроллит к нему. Текущий билет подсвечивается.

### 4.2 Чтение

- Рендер `bodyHtml` каждого билета. Заголовок билета = «Билет N. Название» + блок подпунктов (вопрос) сверху, визуально отделённый.
- Каждый билет — сворачиваемый блок (`<details>` или своя реализация) + кнопки «развернуть всё / свернуть всё» в пределах раздела.
- Мнемоники и ловушки — цветными плашками (см. §6).
- Якоря на каждый билет (`#ticket-4`) для прямых ссылок и для перехода из поиска.
- Кнопка «отметить выучено» прямо в шапке билета (синхронизируется с прогрессом карточек).

### 4.3 Карточки (флэшкарты)

- **Лицо карточки:** «Билет N. Название» + подпункты (вопрос). **Оборот:** тело ответа.
- Карточки генерируются **автоматически из билетов** (1 билет = 1 карточка).
- Тап/клик или пробел — переворот. Кнопки **«Знаю» / «Не знаю»**.
- **Фильтр набора:** по разделу, по конкретному билету или «всё».
- **Перемешивание** набора (кнопка «Перемешать»).
- **Простой spaced repetition (система Лейтнера, 3 коробки):**
  - новая → коробка 1; «Знаю» → +1 коробка; «Не знаю» → сброс в коробку 1.
  - в очереди карточки из меньших коробок показываются чаще; «Не знаю» всплывает скоро.
  - состояние каждой карточки в `localStorage` (`box`, `lastSeen`, `status: new|learning|known`).
- Счётчик «осталось / знаю / не знаю» в текущей сессии.
- Десктоп: стрелки ←/→ — следующая/предыдущая, пробел — переворот.

### 4.4 Тесты (MCQ)

- Вопросы берутся из `content/quiz.json` (схема в §5).
- **Запуск теста:** выбор области (раздел / билет / всё) и количества вопросов (10/20/всё).
- По одному вопросу: текст + 4 варианта (один верный). После ответа — подсветка верного/неверного + короткое пояснение (`explain`).
- В конце — **результат** (N из M, % по разделам) и **разбор ошибок** (список вопросов, где ошибся, со ссылкой на соответствующий билет в Чтении).
- История прохождений в `localStorage` (дата, область, результат) — показывать на Главной.

### 4.5 Поиск

- Индекс по `title` + `subpoints` + `plainText` всех билетов.
- Мгновенная фильтрация по мере ввода (debounce ~150 мс), подсветка совпадений.
- Клик по результату → Чтение, скролл к билету. Можно простую реализацию без внешних либ; если нужна, допускается вендорить `fuse.js` локально.

### 4.6 Прогресс (Главная)

- % билетов со статусом «знаю».
- Прогресс по разделам (полоски).
- Последние результаты тестов.
- Кнопка «Сбросить прогресс» (с подтверждением).

## 5. Тесты: `quiz.json`

MCQ нельзя качественно сгенерить «механически» — варианты-обманки требуют понимания материала. Поэтому:

- **Сгенерируй `quiz.json`, опираясь на содержание `konspekt.md`** (прочитай тело ответов и составь осмысленные вопросы с правдоподобными дистракторами).
- Цель — **3–5 вопросов на каждый билет**, который есть в конспекте. Если объём велик — сделай качественно для всех имеющихся билетов; файл структурируй так, чтобы его легко дополнять.
- Дистракторы должны быть правдоподобными (из той же области: соседние паразиты, похожие термины, близкие цифры), а не абсурдными.

**Схема:**

```json
[
  {
    "id": "q4-1",
    "ticket": 4,
    "section": "Цитология",
    "question": "Кто сформулировал основные положения клеточной теории?",
    "options": ["Шлейден и Шванн", "Вирхов", "Левенгук", "Гук"],
    "answer": 0,
    "explain": "М. Шлейден (1838) и Т. Шванн (1839) — авторы клеточной теории; Вирхов добавил принцип «каждая клетка из клетки»."
  }
]
```

`answer` — индекс верного варианта в `options`. Провалидируй JSON и уникальность `id`.

## 6. Дизайн / UX

- **Чистый, академический, читаемый.** Не перегружать. Главное — текст конспекта легко читать с телефона.
- Светлая и тёмная темы (переключатель; уважать `prefers-color-scheme`).
- Типографика: системный стек с хорошей кириллицей; крупный комфортный шрифт для тела; межстрочный интервал ≥ 1.6.
- Цветовые роли (через CSS-переменные):
  - ключевые факты/`**жирный**` — акцентный цвет;
  - плашка **определения** — нейтральный фон;
  - 🧠 **мнемоника** — мягкий «тёплый» фон;
  - ⚠️ **ловушка** — «предупреждающий» фон.
- Sticky-навигация сверху; на мобиле — нижний таб-бар для 4 режимов и гамбургер для списка билетов.
- Анимации лёгкие; уважать `prefers-reduced-motion`.
- Доступность: семантические теги, фокус-стили, контраст, навигация с клавиатуры.

## 7. Деплой на Cloudflare Pages

Прямой upload папки `public/` (Git не нужен). В сессии уже есть `CLOUDFLARE_API_TOKEN` и `CLOUDFLARE_ACCOUNT_ID`.

```bash
# один раз — создать проект Pages
wrangler pages project create bio-exam --production-branch=main

# деплой (повторяй после каждого изменения контента/кода)
wrangler pages deploy public --project-name=bio-exam
```

После деплоя выдаётся ссылка вида `https://bio-exam.pages.dev` — открывается с телефона.

Локальная проверка перед деплоем (важно: `localStorage` и `fetch` markdown не работают по `file://`):

```bash
wrangler pages dev public      # или: npx serve public
```

В `README.md` опиши: как заменить `content/konspekt.md`, как дополнить `content/quiz.json`, команду редеплоя.

> Деплой публикует сайт. Запускай команду деплоя сам/по моему явному подтверждению — не раньше, чем сайт собран и проверен локально.

## 9. Образец `content/konspekt.md` (для разработки, если настоящего ещё нет)

```markdown
# Цитология

## 4. Клеточная теория
> а) История формирования представлений о клетке.
> б) Основные положения клеточной теории (Шлейден, Шванн).
> в) Развитие клеточной теории в работах Вирхова.

### Основные положения
- **Клетка** — элементарная структурная и функциональная единица живого.
- Клетки сходны по строению и обмену веществ (гомология).
- Размножение клеток — путём деления исходной (**Вирхов:** «каждая клетка из клетки»).
- Многоклеточный организм — система клеток, объединённых в ткани и органы.

🧠 «**Шлейден растил растения, Шванн — животных, Вирхов всех делил**» — авторы и вклад.

⚠️ Не путать: Шлейден и Шванн (1838–39) сформулировали теорию, а принцип деления клеток добавил **Вирхов (1858)**.

# Медицинская паразитология

## 86. Медицинская паразитология
> а) Предмет и задачи медицинской паразитологии.
> б) Понятия «паразит», «хозяин».
> в) Классификация паразитов и хозяев.

**Паразитизм** — форма межвидовых отношений, при которой один организм (**паразит**) использует другой (**хозяина**) как среду обитания и источник питания, нанося ему вред.

- **Хозяева:** окончательный (половозрелая стадия / половое размножение), промежуточный (личиночные стадии), резервуарный.
- **Паразиты:** по локализации — эктопаразиты и эндопаразиты; по времени — временные и постоянные; по обязательности — облигатные и факультативные.

## 104. Бычий цепень
> а) Систематика и морфология.
> б) Цикл развития.
> в) Лабораторная диагностика и профилактика тениаринхоза.

**Бычий цепень** (*Taeniarhynchus saginatus*), класс Ленточные черви — возбудитель **тениаринхоза**.

- **Морфология:** длина 4–10 м; сколекс с **4 присосками, без крючьев** (невооружённый); в зрелой проглоттиде матка с **17–35 боковыми ответвлениями**.
- **Цикл:** окончательный хозяин — человек; промежуточный — **крупный рогатый скот**. Финна — **цистицерк**. Заражение человека — при поедании сырого/недожаренного говяжьего мяса с финнами.
- **Диагностика:** обнаружение зрелых проглоттид (активно выползают) и яиц в фекалиях. **Профилактика:** не есть сырую говядину, ветсанэкспертиза мяса.

🧠 «**Бычий — без крючьев, много веточек (17–35); свиной — с крючьями, мало (7–12)**».

⚠️ Яйца бычьего и свиного цепней **неотличимы** под микроскопом — различают по строению зрелых проглоттид/сколекса.
```
