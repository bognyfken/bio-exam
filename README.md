# Биология · тренажёр к экзамену

Статический сайт-тренажёр для подготовки к экзамену по биологии (Лечебное дело / Педиатрия,
126 билетов). Парсит конспект из markdown **прямо в браузере** и строит 4 режима:
**Чтение · Карточки · Тест · Поиск** + Главная с прогрессом.

Без бэкенда, без фреймворков. `marked` вендорен локально (`public/vendor/`) — сайт работает
офлайн после первой загрузки. Прогресс — в `localStorage`. Деплой — Cloudflare Pages.

## Структура

```
public/                 ← деплоится на Pages целиком
├─ index.html
├─ styles.css
├─ js/                  ← ES-модули (parser, store, режимы, app)
├─ vendor/marked.min.js ← markdown→HTML, локально
└─ content/
   ├─ konspekt.md       ← ИСТОЧНИК конспекта
   └─ quiz.json         ← банк вопросов для тестов
wrangler.toml
```

## Формат `content/konspekt.md`

Парсер (`public/js/parser.js`) опирается на реальную разметку конспекта:

| Элемент | Разметка |
|---|---|
| Документ (игнор) | `# Заголовок` |
| **Раздел** | `## РАЗДЕЛ I. НАЗВАНИЕ` |
| **Билет** | `### N. Название` |
| Подпункты вопроса | `**а) Заголовок.** текст…` (жирный лид-ин в начале строки) |
| 🧠 Мнемоника | строка, начинающаяся с `🧠` |
| ⚠️ Ловушка | строка, начинающаяся с `⚠️` |

> Это отличается от образца в брифе (там разделы = H1, билеты = H2, подпункты = blockquote).
> Парсер написан под **реальный** файл — он источник правды.

## Формат `content/quiz.json`

Массив вопросов MCQ:

```json
[
  {
    "id": "q4-1",
    "ticket": 4,
    "section": "Цитология и молекулярная биология",
    "question": "Кто сформулировал клеточную теорию?",
    "options": ["Шлейден и Шванн", "Вирхов", "Левенгук", "Гук"],
    "answer": 0,
    "explain": "Шлейден (1838) и Шванн (1839); Вирхов добавил «каждая клетка из клетки»."
  }
]
```

`answer` — индекс правильного варианта в `options`. `id` уникален.

## Обновление контента

Код **не зависит** от конкретных билетов. Чтобы обновить:

1. Заменить `public/content/konspekt.md` (тот же формат) — билеты/разделы подхватятся сами.
2. Дополнить `public/content/quiz.json` новыми вопросами.
3. Передеплоить (см. ниже).

## Локальный запуск

`localStorage` и `fetch` markdown **не работают по `file://`** — нужен сервер:

```bash
npx serve public          # http://localhost:3000
# или
wrangler pages dev public
```

## Деплой

### Основной хостинг — GitHub Pages (доступен в РФ без VPN)

Живой адрес: **https://bognyfken.github.io/bio-exam/**

Репозиторий: `bognyfken/bio-exam`. `main` — исходники, ветка `gh-pages` — собранный
сайт (содержимое `public/` в корне). Pages раздаёт ветку `gh-pages` / `/`.

Передеплой после изменения контента/кода:

```bash
git add -A && git commit -m "…"        # фиксируем в main
git push origin main
git subtree push --prefix public origin gh-pages   # выкатываем public/ в gh-pages
```

> Токен GitHub без scope `workflow`, поэтому используется ветка `gh-pages`,
> а не GitHub Actions. При изменении ассетов/контента — поднять `VERSION` в
> `public/sw.js`, иначе у установивших PWA останется старый кэш.
